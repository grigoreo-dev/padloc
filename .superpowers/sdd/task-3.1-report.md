## Verification

Initial flag check:

```text
packages/pwa/package.json:46:        "build": "NODE_OPTIONS=--openssl-legacy-provider webpack",
packages/pwa/package.json:47:        "dev": "NODE_OPTIONS=--openssl-legacy-provider webpack serve",
packages/extension/package.json:45:        "build": "NODE_OPTIONS=--openssl-legacy-provider webpack"
```

After script edits, `pnpm run pwa:build && pnpm run web-extension:build` initially failed in PWA on Node 18 with `ERR_OSSL_EVP_UNSUPPORTED`. Investigation showed webpack 5.52.0 still called `crypto.createHash("md4")` internally from `ChunkGraph`, even though `packages/pwa/webpack.config.js` already had `output.hashFunction: "sha256"`.

Final build command:

```bash
export NVM_DIR="$HOME/.nvm"; . "$NVM_DIR/nvm.sh"; nvm use 18 >/dev/null; corepack prepare pnpm@10.15.0 --activate >/dev/null 2>&1; pnpm run pwa:build && pnpm run web-extension:build
```

Final build output summary:

```text
@padloc/pwa build: webpack 5.61.0 compiled successfully in 58360 ms
@padloc/extension build: webpack 5.61.0 compiled successfully in 40400 ms
```

Both builds emitted the existing webpack deprecation warning for `Compilation.assets`; no build failure.
